# System i18n (internacjonalizacja)

## Architektura

Dwupoziomowy system tlumaczen:

1. **Statyczne tlumaczenia UI** — pliki JSON (`messages/pl.json`, `messages/en.json`)
2. **Dynamiczne tresci z DB** — pola `contentPl` / `contentEn` w tabeli `Section`

## Jak dziala

### Provider

`I18nProvider` (w `lib/i18n.ts`) opakowuje aplikacje i udostepnia:
- `locale` — aktualny jezyk (`'pl'` | `'en'`)
- `setLocale(lang)` — zmiana jezyka (zapis do localStorage)
- `t(key)` — tlumaczenie klucza z plikow JSON

### Uzycie w komponencie

```tsx
import { useLocale } from '@/lib/i18n';

function MyComponent() {
  const { locale, t } = useLocale();
  return <p>{t('reservation.title')}</p>;
}
```

### Dynamiczne tresci z DB

W `HomeClient.tsx` helper `c(section, field)` pobiera wartosc pola wg aktualnego jezyka:

```tsx
const c = (section, field) => {
  const content = locale === 'pl' ? section.contentPl : section.contentEn;
  return content[field] ?? '';
};
```

## Format plikow tlumaczen

Pliki `messages/pl.json` i `messages/en.json` — plaski obiekt klucz-wartosc:

```json
{
  "menu.koncept": "koncept",
  "reservation.title": "Zarezerwuj swoj czas",
  "section.readMore": "CZYTAJ WIECEJ"
}
```

Konwencja kluczy: `sekcja.nazwa` (np. `menu.koncept`, `footer.corporate`).

## Przelacznik PL/EN

W `TopMenu.tsx` — dwa linki PL / EN. Aktywny jezyk ma klase `is-active`.
Wybor jezyka zapisywany w `localStorage` (klucz: `hommm_locale`).

## Jak dodac nowy jezyk

1. Utworz `messages/XX.json` (np. `messages/de.json`)
2. W `lib/i18n.ts`:
   - Dodaj `import deMessages from '@/messages/de.json'`
   - Rozszerz typ `Locale`: `'pl' | 'en' | 'de'`
   - Dodaj do `messagesMap`: `de: deMessages`
3. W `TopMenu.tsx` dodaj link z nowym jezykiem
4. W sekcjach DB dodaj odpowiednie pola (lub rozszerz schemat Prisma o nowe kolumny)
